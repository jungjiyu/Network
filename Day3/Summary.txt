네트워크의 주소 체계
	1) MAC address 물리적 주소
		: 컴퓨터는 네트워크 상에서 정확한 통신을 위해 서로 구분하기 위하여 MAC 이란 "고유한"주소를 가진다	
			: 그러니까 컴터의 주민등록번호 같은 느낌(전세계에서 유일)
			: 인터넷이 가능한 모든 장치에 존재 (스마트폰, 홈 cctv 포함)

			: 네트워크 장비의 인터페이스(NIC. Router, Switch )도 MAC address 를 가진다 
				:NIC
					: Network Interface Card 를 의미. 
					: LAN 카드란 단어는 사실 존재하지 않음. 뭐 일본의 영향을 받아 한국서 통상 자주 쓰긴 하지만. 사실 LAN 카드가 아니라 NIC 카드라고 부르는게 맞는거임


		: 00-19-D1-F0-09-FF , 00:19:D1:F0:09:FF , 0019.D1F0.09FF 같은 형태
		: (이진수로는 48bit인데) 16진수로 표현되어 총 12자리로 표현됨
			: 구성 
				OUI Organizational Unique Idnetifier : 앞의 6개의 16진수(=24bits)는 생산자(=회사메이커)를 나타내는 코드
				Host identifier : 뒤의 6개의 16진수(=24bits) 회사에서 각 장비에 분배하는 시리얼넘버

	2) IP address 논리적 주소
		: 10진수로 표현한 주소 체계 (2진수론 32bit)
		: 192.178.21.1 같은거

	: TCP/IP Protocol 을 사용하는 네트워크 (internet 같은거) 에서는 IP address 를 사용하여 통신하나, 최종적으론 MAC address 를 사용하여 데이터를 전달함
        	:  전화를 해도 "길동이 맞나요?" 라고 묻는건 전화받는 사람이 길동인지가 확실하지 않기 때문, 이와 비슷하게 ip 주소로 데이터를 실어서 보내고 실제로 데이터를 받아야하는 기계가 맞는지 MAC address 를 확인하고 전달해줌.
		:TCP/IP Protocol
			: 목적지주소에다가 데이터 실어서 보내는 방식
