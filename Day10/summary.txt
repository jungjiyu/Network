*** 4. 192.168.0.0(/24) , A팀=100명 , B팀=60명 , C팀=25 명 , D팀=10명, E팀 = 5명 으로 나눠라
	: 그냥은 각 팀별로 크기를 다르게 하여 서브넷을 나눌 순 없다.
	: VLSM( Variable Length Subnet Mask )
		: 가변 길이 서브넷 마스크
			: 기존의 고정 길이 서브넷 마스크보다 효율적으로 ip 를 윤영, ip 주소 공간의 낭비를 줄일 수 있다
			: 복잡한 네트워크 구현
			: 여러개의 작은 네트워크로 분할하여 네트워크 보안정책을 강화할 수 있다.


		: 하는 방법
			: 가장 큰 호스트의 크기를 갖는 네트워크부터 점점 작은 크기의 호스트를 갖는 네트워크 순으로 서브넷팅을 한다(네트워크를 나누는게 아니라)
				: 뒷순서로 구한 구간의 경우 앞 순서 구하고 남은 범위에서 구하는 거다
			: 그러니까 ip 는 하난데 팀마다 서브넷 마스크가 달라진다

	: A팀
		: sm >> 255.255.255.128
		: 구간>>192.168.0.0 ~192.168.0.127 // 총 128 개

	: B팀
		: sm >> 255.255.255.192
		: 구간>>192.168.0.128 ~192.168.0.191 // 총 64 개
			: A 팀 나눠주고 남은 구간에서 구하는 것 주의
	: C 팀
		: sm >> 255.255.255.224
		: 구간>>192.168.0.192 ~192.168.0.223 // 총 32 개
			: B팀 까지 나눠주고 남은 구간에서 구하는 것 주의

	: D 팀
		: sm >> 255.255.255.240
		: 구간>>192.168.0.224 ~192.168.0.239 // 총 16 개
			: C팀 까지  나눠주고 남은 구간에서 구하는 것 주의


	: E팀
		: sm >> 255.255.255.248
		: 구간>>192.168.0.240 ~192.168.0.248 // 총 8 개
			: D팀 까지  나눠주고 남은 구간에서 구하는 것 주의



5. 192.168.0.0(/24) , 500명 
	: CIDR ( Classless Inter-Domain Routing ) - 사이더(싸이더)
		: 1993년 도입된 최신의 ip 주소 할당 방법
		: 기본의 클래스를 무시하고 비트 단위로 재할당
		: 호스트의 크기에 맞게 효율적으로 운영이 가능해짐

		: ip 주소는 한정되어있으니까 원하는 크기의 ip가 없을 경우,  남아있는 ip의 기존 클래스를 무시하고, 다시 할당함
	
	: SM >> /24에서 255.255.254.0 으로 서브넷 마스크를 바꿈
	: 구간>> 192.168.0.0 ~192.168.1.255
