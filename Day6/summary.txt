OSI 7 layer, TCP/IP 모델 비교
	: 공통점 >> 둘다 계층형이다
	: 차이점 
		>> 각 계층이 서로 상이하다
		>> OSI 7 layer 는 가이드 역할로 충실하지만 실제 구현의 예가 거의 없어 신뢰도가 약하다
		>> TCP/IP 모델은 인터넷 개발 이후 계속 표준. 수십년간 ㅇ운용 경험에 의해 신뢰도가 높다

TCP/IP 참조 모델
	: 인터넷의 전반적 과정을 다룬 모델
	: OSI 7 Layer 보다 더 먼저 나온 현재에도 가장 범용적으로 사용하는 모델이다
	: OSI 7 Layer 모델이랑 분명히 다른 거
	: 계층
		1계층 Network Interface
			: OSI 7 layer 의 Phisycal Layer, Data Link Layer 에 해당
		2계층 Internet Layer
			: OSI 7 layer 의 Network Layer 에 해당
		3계층 Transport Layer
			: OSI 7 layer 의 Transport Layer 에 해당
		4계층 Application Layer
			: OSI 7 Layer 의 Session Layer , Presentation Layer, Application Layer 에 해당

OSI 7 Layer 모델
	: 통신이 일어나는 과정을 전반적으로 다룬 모델로 TCP/IP 참조 모델을 세분화 함
	: 국제 표준화 기구(ISO)가 1984년에 발표
	: 1>>2>>,...>>7 계층 방향으로만 진행되는게 아니라 7>>...>>1 방향으로도 일어난다
	: 계층별 통신단위, 장비는 나오니까 꼭 외우자										
	: 계층
		1계층 - Physical Layer 물리계층
			:  전기적, 기계적인 특성을 이용해서 통신 케이블로 데이터를 전송하는 계층(물리적 장비)
			: 단지 데이터를 전달"만" 하고 받으려는 데이터에 대한 에러에 대한 검사는 안함
			: *** 단위) 비트 단위(= 0 , 1 로 나타나진다 = 전기적으로 on off 다)
			: *** 대표 장비) 통신케이블,리피터, 허브



		2계층 - Data Link Layer 데이터 링크 계층
			: 물리계층을 통해 송수신되는 데이터의 오류와 흐름을 관리하여 신뢰성 제공 계층
				: 신뢰성 >> 대상을 구분한다
			: 1계층이 물리적 계층이었다면 2계층은 논리적 계층이다.
			:*** 단위 )프레임
				: 2 계층에서 데이터를 전달하는 단위를 걍 프레임이라 함
				: 프레임 단위로 MAC 주소를 부여하여 통신
					: MAC 주소를 사용하지만 ARP 가 2계층에 해당하지 않는 것은 ip 주소 정보가 없기 때문
			: ***대표 장비) 브릿지, 스위치, 이더넷
 

		3계층-네트워크 계층 Network Layer
			: 빠른 길을 찾는 개념
			: 라우팅 프로토콜을 이용해서 데이터를 목적지까지 최대한 안전하고 빠르게 전달하는 계층
				: IP, IPX담당 + 라우팅기능(여러개의 노드를 거칠때마다 경로를 찾아줌)
			:*** 단위) 패킷 packet 
				: 3계층에서 데이터를 전달하는 단위를 걍 패킷이라고 함
			: ***대표장비) 라우터

		4계층-전송개충 Transport Layer
			: 어떤 방식으로 전달할지 정하는 개념(버스를 탈꺼야 택시를 탈꺼야?)
			: 정보를 분할하고, 상대편에 도달하기 전에 다시 합치는 과정을 담당
				: 큰 통나무를 그냥 배송하면 통나무에 상처 너무 많이남(중간에 터널에 부딪히고 뭐하고). 차라리 일정하게 통나무를 잘라서 배송하고, 목적지 도달이전에 하나로 합침
				: port 번호와 전송방식(TCP || UDP) 결정 하여 TCP 헤더를 붙여서 두 지점간의 신뢰성 있는 데이터를 주고 받게 함

			: ***단위) 세그먼트
			: ***통신 장비) 게이트웨이
			: ***4계층 프로토콜) TCP , UDP
				: TCP >> 신뢰성, 연결지향적
					: 목적지까지 안정적으로 도달해야하는 성향의 통신
					: 하위카테고리 >>  http, hppts, ssh 등등
				: UDP >> 비신뢰성, 비연결성, 실시간
 					: 목적지까지 데이터를 빠르게 전송만 하면 되는 성향의 통신
					: 하위카테고리 >> dhcp, dns, 스트리밍 서비스 등등

		5계층- 세션 계층 Session Layer
			: 세션을 생성 및 종료
				: 그러니까 다운로드 받던거 중지하고 끄고, 나중에 그거 다시 다운로드 받으면 이미 다운로드 받았던 부분까지는 빠르게 된다.
			
			: API, Sokcet
		
		6계층-표현계층 Presentation Layer
			: 전송하는 데이터의 표현 방식(데이터변환 압축 ,암호화등 인코딩 방식)을 결정
				: 암호화 >> 데이터를 알아볼 수 있게 만드는것
					: 암호화 예시>> vpn
						: 
			: JPG , GIF, ASCII 등

		7계층-응용계층 Application Layer
			: 최종 목적지. 응용 프로세스와 직접 관계하여 일반적인 응용 서비스르 수행(explore, chrome)
			: 대표 프로토콜) HTTP , FTP, IMAP 등

 NIC >> 네트워크 인터페이스 카드. 컴퓨터를 네트워크 상으로 통신 가능하게 해주는 하드웨어 장치
통신의 방향성>>
	1. simplex
		: 단방향 전송 방식.  데이터의 수신만 가능(=response를 못한다 )
		: 실시간 방송
	2. half duplex
		: 반이중 전송 방식. 데이터의 수신과 송신이 모두 가능하지만 동시에 수신과 송신을 할 순 없다
		: 무전기
	3. full duplex
		: 전이중 전송방식. 데이터의 수신과 송신을 동시에 할 수 있다.
		: 전화기

Reapeater 와 Hub 와 Switch 와 Router 와 Gateway
	Repeater 리피터
		: 기능
			1.감쇄된 신호를 (초기화한 후 재생산해서) 증폭
			2. 케이블의 연장
		: 요즘에는 잘 안씀. 건물에 이미 설비가 다 되서 들어오기 때문. 

	Hub
		: 기능
			1. 감쇄된 신호를 (초기화한 후 재생산해서) 증폭
			2. 여러대의 장치를 연결 << 멀티포트 기능이라고 한다
		: 문제점
			: 자체적인 메모리가 없어 패킷의 목적지를 알지 못하여, 브로드캐스팅 방식으로 연결되어있는 "모든" 장치에게 전달한다	
			: half duplex 이다 >> collision 자주 일어난다

	switch
		: 기능
			1. MAC주소로 식별하여 연결되어있는 특정 로컬 장치에게 전달
		: LAN 방식
		: 작동 원리
			0. ARP 과정을 통해 연결된 장치의 MAC 을 연결된 포트의 번호와 맵핑하여 메모리에 저장해둠
				: ARP 과정은 Hub의 작동 방식과 비슷하게 브로드 캐스팅으로 진행됨. ARP 과정 이후론 유니캐스트.
			1. 연결되있는 기기가 타기기로 패킷 보냄
			2. 중간에 낀 스위치는 송신하는 기기가 보낸 패킷의 헤더를 읽어 MAC 주소를 확인
			3. MAC 주소로 구분하여 목적지로로 패킷 보냄

		: 대부분 full-duplex (전이중) 기능을 제공

				

	router	
		: 기능
			1. 내부 네트워크와 외부 네트워크를 연결해줌
			2. 데이터들이 전송되는 최적의 경로 결정
				: 최적의 경로 결정 방법
					(1) 다이나믹 라우팅 방식 >> 자동적으로 경로 설정
						sol1 >> Distance Vector - 홉카운트만을 가지고 최적 경로를 연산
							: 홉카운트 >> 경유하는 라우터의 개수
							: 주로 소규모 네트워크에서 사용
							: 비유 ) 역이 더 적어서 먼저 종착지에 도착할꺼다 
						sol2 >> Linkstate - 장치간의 속도 등 전반적인 정보를 종합해서 최적의 경로를 연산
							: 주로 대규모 네트워크 에서 사용
							: 비유) 지하철 속도가 더 빨라서 먼저 종착지에 도착할꺼다

					(2) 스태틱 라우팅 방식 >> 사람이 일일이 경로를 미리 설정해둠

			3. NAT 기능
				: 사설 IP 를 공인 IP 로 바꿔준다
		: WAN 방식
		: 라우터에 스위치 기능이 내장되있는 경우도 있다
		: 가정집에 있는 공유기는 가정용 라우터이다. (기업용 라우터도 따로 존재)


	gateway	
		: 라우터처럼 장비가 아니라 "개념적" 의미.
		: 내부 네트워크에서 다른 네트워크로 가기 위해 반드시 거쳐야하는 통로. 접속지점.
