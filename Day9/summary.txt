문제1) 192.168.0.3  일때 20명 근무 한다. 서브넷팅해라
	: 1 2 4 8 32 ... 중에서 20과 가장 가까운게 32 이므로 거기까지 1 표시해서 networkID 화 한다
	: 서브넷 마스크가 딱히 주어지지 않았으므로 default subnetmask 를 사용한다

	: 서브넷팅 고려사항
		: 피자조각의 크기와 개수

****문제8) 192.168.0.110, 20명
	: default subnetmask ) 255.255.255.0
	: 서브넷 마스크 ) 255.255.255.224
	: 서브넷 주소 ) 192.168.0.96 ~192.168.0.127 // 32개
		: 마지막 한 옥텟 중 앞 3 개까지는 networkID 이고 남은 5조각은 hostID 인데, hostID 부분의 5개의 비트로 표현할 수 있는 수이 범위는 다음과 같다 다음의 조각 중 110은 96~127 범위에 속한다.
			: 나눠지는 범위의 개수는 2** 3으로 , 가능한 서브넷의 개수와 같고 , 각 범위의 시작은 젤 끝 1이 위치하는 값인 32의 배수이다.
				: 각 범위의 시작 값이 젤 끝 1 이 위치하는 값의 n배가 되는 것은 그게 가능한 최소 단위이기 떄문이다
			0~31  // 001 00000 ~ 001 11111
			32~63  // 010 00000 ~ 010 11111
			64~95  // 011 00000 ~ 011 11111
			96~127  // 100 00000 ~ 100 11111
			128~159
			160~191
			192~223
			224~225 

	: 가용 ip 주소 ) 192.168.0.95 ~192.168.0.126
