인트라넷
	: 회사에서 쓰는 여러가지 프로그램을 인터넷을 사용하는 것처럼 쓰도록 만들어 논것. 내부의 네트워크(회사 직원 외엔 사용 불가)
		: LG 에서 사용하는 인트라넷. 외부에서 그 dns 를 입력하면 그 사이트 없다고 뜸. 이는 사내에서 운영하는 dns 서버를 거쳐 특정 ip 를 get 해야하기 때문
			: 출장 가면 vpn 신청해서 접속할 수 있게 해야된다
				vpn >> 가상 회선으로 외부에서 이용할 수 있게 만들어주는 것
	: 인터넷 없던 시절엔
	: 인트라넷 역시 TCP/IP 프로코콜을 사용
		: TCP, UDP >> 데이터를 보내기 위해 사용하는 프로토콜
			: Transmission Control Protocol
				: 전송 제어하는 프로토콜이란 뜻
				: 인터넷 상에서 데이터를 메세지 형태로 보내기 위해 IP와 보통 함께 사용하는 "연결형" 프로토콜	
					: TCP 는 패킷을 추적및 관리, IP 가 데이터의 배달을 처리한다고 봄 됨
						: 패킷 Packet
							: 데이터 조각들. 
							: 라우팅(=인터넷에서 데이터를 보내기 위한 경로배정)을 효율적으로 하기 위해 데이터를 패킷의 형태로 조각내서 전달함
						: 어떻게 패킷을 추적 밑 관리> 패킷에 번호를 부여한다. 그래서 ㅁ모든 패킷이 도착할 수 있게 하고 , 순차적으로 조립할 수 있게 한다.
					: 연결형 방식으로 패킷을 교환한다. (가상 회선 방식이 아니라.)
						: 논리적 경로를 배정하여 정확히 데이터를 전달한다

						: 1: 1 로 연결한다>> 신뢰성있다

				: 인터넷 환경서 기본으로 사용됨
				: 신뢰성이 필요한 서비스(파일 전송과 같은)에 쓰임

			: UDP User Datagram Protocol
				: 데이터 크램 프로토콜 규약, 즉 데이터를 데이터그램 단위로 처리하는 "비연결형" 프로토콜
					: 데이터그램
						: 독립적인 관계를 지니는 패킷.
					: 비연결형
						: 논리적 경로가 없고, 각각의 패킷은 다른 경로로 전송되어 각 패킷은 독립적 관계를 지니게 됨. 
					: 1:1 , 1:n , n:n 등으로 연결가능
				: 정ㅂ보를 주고 받을떄 정보를 보내거나 받는다는 신호절차 안거침
				: 신뢰성보다 연속성이 중요한 서비스 (스트리밍 같은거)에 자주 쓰임
				: 흐름제어가 ㅇ없어서 패킷이 제대로 전송되었나 알수가 없음
					: 흐름제어 Flow Control 
						: 데이터를 송수신하는 곳의 데이터 처리 속도를 조절해서 수신자의 버퍼 오버플로우를 방지하는 것.(그러니까 문자 ㅈㄴ 보내면 받는 쪽에서 문제 생기잖아)
					: 혼잡제어 Congestion Control
						: 네트워크 내의 패킷 수가 넘치지 않도록 방지하는 것
